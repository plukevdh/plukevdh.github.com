Finally made the first of many updates to my site. Color scheme change is next. Or maybe the reskinning of this blog?

In any case, I’m now with webbynode. I figured if anything, my site ought to be the first one to make a switch as I’m getting a little discontent with Heroku’s availability some days. Moreover, their charges for additional services are getting outrageous. 

It’s not incredibly difficult to setup. Definitely some changes over Heroku, who I must say still have one of the best deploy workflows out there. But what is incredibly valuable is I now have root access to the server, which means I can do just about anything I want to it. I can kill it in cold blood if I want. Big. Fat. “shutdown -t now”, But lets not get hasty…

The big benefit is I can now run Node.JS, Redis, MongoDB, custom ruby tasks, cron jobs… anything I want… for free. Heroku is charging an arm and a leg for that privilege. 

So first is migrating everything from heroku. Not hard. Webby is not as well documented, but its not too hard using the preconfigured Linux stacks. They even have Rails 3 (in fact I hear they were one of the first web hosts to support the beta). 

A few minor complaints thus far. You can’t include your database.yml in the lineup or it complains. Also when using mysql, you are forced to use the old mysql gem rather than the shiny new mysql2 gem which has a lot going on. The fix for this currently is just to update it by hand post-install.

Besides that and some trouble figuring out how to ssh into the thing (simply due to lack of documentation), hungerandthirstdesign.com is running like a champ. Moreover, I can now run some other images processors (setup to thumbnail images for Peerdrum.com) that I had running on an EC2 instance over here now, which is pretty freaking awesome since EC2 costs me $60/mo. to run with no real usage out of it. So that’s a big deal.

Because of my work supporting Hargray’s custom linux rails server stack, I know how to do custom scaling and failover for these machines and while having Heroku’s slider based scaling system is nice, doing it my own way is nicer. 

New experiment too: Trying on Nginx for the first time. Running passenger on top of that for now. Seems like a really clean way to do web server configuration. I like. Seems fast enough. Consumes very little memory which is good on my lower-end node.

So all in all, I like Webbynode so far. I’ll be interested to start integrating more stuff into it like Redis or some other db backend. I guess in a sentence to sum it up: root access is awesome.
