So I mentioned last week that I was making the switch to VIM. Well here we are, a week later, and where are am I so far? Much better than expected. I installed fresh VIM on my linux dev machine and, since I haven’t found a dependable editor on linux that wasn’t slow as hell or almost totally useless for my customary workflow, it was fairly easy not to be tempted to revert to my old editor, since I didn’t really have one on Linux. Mac still pulls me in the way of TextMate or Coda (which is still one of my favorites of all time), but even there, for quick edits, and even hardcore refactoring, its getting really hard to be without VIM’s super-extensive and extendable keyboard shortcuts.

It’s definitely a learning curve and I’m still very much learning and trying to remember what does what and what will make this function faster, how I can bounce about, replace, update code better. It takes a lot of memory power, but starting out slow, you kinda just learn as you go along. :h gives me plenty of info about things I’ve seen people do or want to learn how to do. The built in help is probably one of the best things ever. Every editor should come with this much help before it can be released. I’m srsly you guys…

I’m generally a fan of one input device if I can help it, and this is definitely where VIM shines for me. Even in MS Word, I start typing :w to save instead of Ctrl-S… But now that I’m getting the swing of the navigation tools within VIM, I’m getting pretty versatile in moving around the code really quickly. A few plugins I would recommend using early on (regardless of what language you’re working with) to aid in the migration are:

Pathogen – Simple plugin installation just by dropping vim plugins into a single folder.

NERDTree – A TextMate-like drawer/file browser for VIM. Splits browsing files into a separate buffer for quick perusal of projects and the like.

BufExplorer – Lets you jump around your buffers very quickly. Not necessary, but I use this one to bounce around files a lot faster. Almost like tabs for your editor (without using VIM’s built-in tabs).

VCSCommand if you use SVN or a wide variety of other SVC systems. Git is included in this pkg, but I like to use vim-git and vim-fugitive instead).

Ack – Better than grep. All you need to know.

There are half a dozen others I use for Rails/Ruby development, but these are the mainstays. I am in VIM to stay for now. At least until TextMate2 comes out…
