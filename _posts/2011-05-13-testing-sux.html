Maybe its my lack of usage, maybe its my lack of desire, but I hate writing tests.

Its not because I’m not good at it or that its hard to do, but it sucks, pure and simple. Why?

  1. It kills motivation to start something. It seems a massive undertaking, especially at the outset of a project to setup a decent testing structure for projects (somebody please make this easier). If it didn’t take knowing how to set up all my frameworks, gems, helpers, factories (or spending hours looking up docs about it), maybe I’d be more inclined, but its just daunting.
  2. It eats up dev time. I like programming, playing with code, tinkering with something until it works. Once it works, I can refactor. I could care less about tests. Granted, it ensures going forward my code works, but I know it works (for now). I tend to to CDT (code-driven testing) because of this.
  3. Often times it requires me to write more code than I did for the initial feature. Do I even need to say why I feel like this is not cool? Why would I want to spend more time writing tests than writing fun code? This code will never even see the light of day!
  4. Running tests eats time. It has been called the compiling of interpreted languages. I hate sitting and waiting for things to run. I will admit, CI helps with this. I like submitting tests, even if they break, to CI so it can run and I can write code.
  5. Frameworks require learning time. There are too many libs with too much complication out there. This contributes to #1 and the difficulty in learning, reading, understanding new code and the test framework that was used to test it. Jasmine for testing Javascript code is an incredibly simple library has been really easy to learn. All testing frameworks should take heed.
  6. Testing logic can get complex. Writing tests to test complex logic can be, well, complex.
  7. Tests require you to write testable code. Sometimes elegant solutions are not easily testable. Depends on how you test (blackbox, whitebox…).

All that said (and I may have more), I test. With the best. Consistency over convenience I guess… It does ensure that your code will continue to work as expected. And that is a valuable thing.

It still sucks.
