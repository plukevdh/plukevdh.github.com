Well 6 weeks worth of work on this baby and we now have a fully self-contained distributable project with slick Rails integration.

The initial release of Kthxbye focused on the machine behind the gem: the queuing processing of jobs. This release is more of a front-end and Rails compatibility release. This gem extracts out a lot of the built-in functionality that I had built around the gem into the primary application this gem was built to enhance. Mostly it involves the work I’ve been doing with Node.js and Socket.IO. It’s really simple code and if you want a look at the Node notification engine, it’s here. There’s a lot of new documentation added to the README file which ought to help you figure out how it works and what all needs to be installed. 

This is also my first experiment in Rails 3 > Railtie gemification. Railtie is really powerful engine an unfortunate few undocumented secrets (like that all generators apparently need to be named “<name>_generator.rb” in order to be picked up by Rails automatically). In any case, this gem is now super easy to integrate into your project that needs delayed job computation and callback notifications.

What’s next? Well, probably more features, a few enhancements, better reporting, and more slick ways to interact with Kthxbye.

Check it out now with “gem install kthxbye”. Enjoy all!
