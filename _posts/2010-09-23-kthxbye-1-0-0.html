Hey all. quite pleased to introduce the “final” version of my latest project, Kthxbye. As I wrote about in the last few blog posts, this gem allows you farm out long running processes that you need a result from to a separate processing engine so that you don’t have to tie up your web-server with long running processes.

This doubtlessly will have many revisions in the future but it took me so long to finalize the initial API that I wanted to solidify something for an initial release. This library is fully tested for Ruby 1.8.7 and 1.9.2. 

It has a pretty slick web front-end too. Lots of useful information to display as well as a live polling feature which refreshes the data on the screen every 2 seconds.

It also gives you good indication as to what the status of the job is, when it’s completed, whether or not it succeeded or failed. This is all available through the API, and therefore, to your app as well.

The interface is built on Sinatra and should be pretty easy to integrate into your Rails app stack (made especially easy in Rails 3).

So how about some comments on the project? This is my first time doing any kind of major unit testing. Definitely will be doing more of this in the future. Incredibly helpful to know when I broke something earlier, caught so many more exceptions and edge cases I might have otherwise missed. Obviously that’s the point of unit testing, but it was cool to see just how helpful it was long term.
